# Интернационализация (i18n) — архитектура, глосар и план

## Архитектура

| Слой | Файл |
|---|---|
| Конфигурация на езиците | `src/lib/i18n/config.ts` |
| Зареждане/сливане на преводи | `src/lib/i18n/messages.ts` |
| Сървърен достъп (`getLocale`, `getT`) | `src/lib/i18n/server.ts` |
| Locale-aware форматиране | `src/lib/i18n/format.ts` |
| Client провайдър/хук (`useT`, `useI18n`) | `src/components/i18n/I18nProvider.tsx` |
| Превключвател на езика | `src/components/i18n/LanguageSwitcher.tsx` |
| Запис на избора (cookie + профил) | `src/app/api/locale/route.ts` |
| Преводи | `src/locales/<lang>/<namespace>.json` |

- Езици: **bg** (по подразбиране, без URL префикс), **en, ru, ro, tr, el**.
- Fallback: липсващ ключ → български; никога не се показва суров ключ.
- Запазване: бисквитка `cda_locale` (+ `localStorage` fallback) и `User.preferredLanguage` при логнат потребител. Смяната не прекратява сесията.

## Как се добавя НОВ ЕЗИК (напр. немски)
1. Добавете ред в `LOCALES` в `src/lib/i18n/config.ts` (`{ code: "de", label: "DE", native: "Deutsch", flag: "🇩🇪", intl: "de-DE", og: "de_DE" }`).
2. Създайте `src/locales/de/*.json` (копие на `bg` и превод).
3. Добавете импортите/блока в `src/lib/i18n/messages.ts`.
Без промяна по компонентите.

## Как се добавя НОВ NAMESPACE
1. `src/locales/<lang>/<namespace>.json` за всеки език.
2. Импорт + ред в `DICTS` в `messages.ts`.

## Фазов план на превода

- **Фаза 1 (този PR):** i18n архитектура + запазване + switcher (навсякъде) + locale-aware форматиране + `Document.language` (вече наличен) + `User.preferredLanguage` + namespaces `common`, `navigation`. Проверено end-to-end.
- **Фаза 2:** auth (login/register/forgot/verify), публичен сайт (home/software/services/accountants/pricing/about/contact) + SEO (hreflang/canonical/OG locale, sitemap per език).
- **Фаза 3:** dashboard + analytics + BI компоненти + KPI/insights (ключове за insight текстовете).
- **Фаза 4:** фактуриране/документи UI + **PDF шаблони** + „Език на документа" селектор (default: език на клиента → потребителя → bg).
- **Фаза 5:** имейл шаблони (по език на получателя) + нотификации (key + payload).
- **Фаза 6:** останали модули (склад, производство, разходи, договори, проекти, служители, HACCP, счетоводни къщи, Super Admin) + validation/toasts.
- **Фаза 7:** финален audit за hardcoded текст + премахване на неизползвани ключове.
